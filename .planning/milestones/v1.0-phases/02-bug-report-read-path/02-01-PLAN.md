Create three model classes and one repository:

**BugReportSummary** (`lib/domain/models/bug_report_summary.dart`):
- Fields: `String id`, `String description`, `String? appVersion`, `String? platform`, `DateTime createdAt`, `String? githubIssueUrl`, `String sourceApp`
- Factory constructor `fromJson(Map<String, dynamic> json)` parsing `created_at` with `DateTime.parse()`
- Getter `String get descriptionPreview` that truncates description to 120 chars with ellipsis if longer

**BugReportDetail** (`lib/domain/models/bug_report_detail.dart`):
- All fields from BugReportSummary PLUS: `String? deviceInfo`, `String? logs`, `String? screenshotBase64`, `String? userId`
- Factory constructor `fromJson(Map<String, dynamic> json)`

**ProductReportCount** (`lib/domain/models/product_report_count.dart`):
- Fields: `String productName`, `int totalCount`, `int unprocessedCount`
- Getter `int get processedCount => totalCount - unprocessedCount`

**BugReportRepository** (`lib/infrastructure/repositories/bug_report_repository.dart`):
- Import `SupabaseConfig` from `infrastructure/services/supabase_config.dart`
- Three methods:

1. `Future<List<ProductReportCount>> getProductCounts(List<String> productNames)`:
   - For each product name, issue two count queries using `CountOption.exact`:
     - Total: `.from('bug_reports').select('id').eq('source_app', name).count(CountOption.exact)`
     - Unprocessed: same but add `.isFilter('github_issue_url', null)` — comment: `// Phase 2 proxy: replace with triage_tag IS NULL in Phase 3`
   - Return list of ProductReportCount

2. `Future<List<BugReportSummary>> getReportsByProduct(String productName)`:
   - Use explicit column projection: `'id, description, app_version, platform, created_at, github_issue_url, source_app'`
   - CRITICAL: Do NOT use `select('*')` — screenshot_base64 averages 166-350 KB per row
   - Filter: `.eq('source_app', productName)`
   - Order: `.order('created_at', ascending: false)`
   - Map results to `BugReportSummary.fromJson()`

3. `Future<BugReportDetail> getReportDetail(String reportId)`:
   - Use `select('*')` — detail view needs all columns including screenshot
   - Filter: `.eq('id', reportId).single()`
   - Map to `BugReportDetail.fromJson()`

Import `package:supabase_flutter/supabase_flutter.dart` for `CountOption`.

Create two ChangeNotifier controllers following the AuthController pattern:

**DashboardController** (`lib/application/controllers/dashboard_controller.dart`):
- Extends `ChangeNotifier`
- Constructor takes `BugReportRepository` (store as `_repository`)
- State fields: `List<ProductReportCount> _counts = []`, `bool _isLoading = false`, `String? _error`
- Public getters for all state fields
- Method `Future<void> loadCounts(List<String> productNames)`:
  - Set `_isLoading = true`, `_error = null`, `notifyListeners()`
  - Call `_repository.getProductCounts(productNames)`
  - Set `_counts` to result, `_isLoading = false`, `notifyListeners()`
  - Catch errors: set `_error = e.toString()`, `_isLoading = false`, `notifyListeners()`

**ReportListController** (`lib/application/controllers/report_list_controller.dart`):
- Extends `ChangeNotifier`
- Constructor takes `BugReportRepository` (store as `_repository`)
- State fields: `List<BugReportSummary> _reports = []`, `bool _isLoading = false`, `String? _error`, `String? _currentProduct`
- Public getters for all state fields
- Method `Future<void> loadReports(String productName)`:
  - Set `_currentProduct = productName`, `_isLoading = true`, `_error = null`, `notifyListeners()`
  - Call `_repository.getReportsByProduct(productName)`
  - Set `_reports` to result, `_isLoading = false`, `notifyListeners()`
  - Catch errors: set `_error = e.toString()`, `_isLoading = false`, `notifyListeners()`
- Method `Future<void> refresh()`:
  - Guard: if `_currentProduct == null` return
  - Call `loadReports(_currentProduct!)`
