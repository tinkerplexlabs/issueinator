```dart
else if (currentTag == TriageTag.issue)
  ListenableBuilder(
    listenable: _syncController,
    builder: (context, _) {
      final isSyncing = _syncController.isSyncing;
      return FilledButton.icon(
        onPressed: isSyncing ? null : () => _syncReport(context),
        icon: isSyncing
            ? const SizedBox(
                width: 16,
                height: 16,
                child: CircularProgressIndicator(strokeWidth: 2),
              )
            : const Icon(Icons.upload),
        label: Text(isSyncing ? 'Syncing…' : 'Sync to GitHub'),
      );
    },
  )
else
  Text(
    'Not synced to GitHub',
    style: Theme.of(context)
        .textTheme
        .bodySmall
        ?.copyWith(color: Colors.grey),
  ),
```

Note: The existing `if (isDuplicate)` and `else if (detail.githubIssueUrl != null)` branches stay exactly as they are. Only the final `else` branch changes to check for `currentTag == TriageTag.issue` first, then fall through to the "Not synced" text for other non-issue tags.

4. Add `_syncReport(BuildContext context)` async method:
   - Capture `ScaffoldMessenger.of(context)` before any await (per established pattern from Phase 3 decisions).
   - Call `_syncController.sync(widget.reportId, _detail!)` where `_detail` is the report detail.
   - Switch on result type:
     - `SyncSuccess`: Show SnackBar "GitHub issue created" with an action to launch the URL. Call `_fetchDetail()` to refresh the detail (so the "View GitHub Issue" chip appears).

     - `SyncError` without re-auth: Show SnackBar with the error message.

5. Keep the existing `_launchGitHubUrl` method — it's already used by the "View GitHub Issue" chip and will be reused for SnackBar actions.
