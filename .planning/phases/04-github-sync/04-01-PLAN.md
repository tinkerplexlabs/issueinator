`syncReport(reportId, detail)` follows this exact flow (from research Pattern "Full Sync Flow"):
1. Get stored token via `_githubAuthService.getStoredToken()`. If null → return `SyncError('Not authenticated', requiresReAuth: true)`.
2. Look up repo via `_repoMap[detail.sourceApp]`. If null → return `SyncError('No GitHub repo mapped for "${detail.sourceApp}"')`.
3. Compute content hash: SHA-256 of `'$description\n$deviceInfo\n$platform\n$appVersion'`, take first 12 hex chars. Use `crypto` package `sha256.convert(utf8.encode(...))`. This MUST match the CLI tool algorithm exactly.
4. Check for existing issue via `_findExistingIssue(hash, repo, token)` — GraphQL search. If found, call `_repository.updateGithubIssueUrl(reportId, existingUrl)` and return `SyncDuplicate(existingUrl)`. On `GitHubAuthException`, revoke token and return `SyncError('GitHub token expired', requiresReAuth: true)`.
5. Fetch screenshot via `_repository.getReportScreenshot(reportId)`. If non-null and non-empty, upload via `_uploadScreenshot(reportId, screenshotB64)`. Non-fatal if upload fails — proceed without screenshot URL.
6. Build issue title: first 80 chars of description (or full if shorter), prefixed with "[Bug] ".
7. Build issue body matching CLI tool format:
   - Metadata table: Platform, Device, App Version, Report ID
   - HTML comment: `<!-- hash:XXXXXXXXXXXX -->`
   - Description section with `## Description` header
   - Screenshot section with `## Screenshot` header and `![screenshot](url)` if URL available
   - Logs section in `<details><summary>Logs</summary>` collapse, truncated to last 512 KB
8. Create GitHub issue via `_createGitHubIssue(repo, title, body, token)`. On success, call `_repository.updateGithubIssueUrl(reportId, issueUrl)` and return `SyncSuccess(issueUrl)`. On `GitHubAuthException`, revoke token and return error.

Private helper methods:

`_contentHash(description, deviceInfo, platform, appVersion)` → String (12 hex chars)

`_findExistingIssue(hash, repo, token)` → Future<String?> — POST to `https://api.github.com/graphql` with query `search(query: "hash:$hash in:body repo:$repo", type: ISSUE, first: 1)`. Parse `data.search.nodes[0].url`. Check response status 401 → throw `GitHubAuthException()`. NOTE: GraphQL returns the web URL in the `url` field of Issue nodes (unlike REST where `url` is the API URL).

`_uploadScreenshot(reportId, base64Data)` → Future<String?> — Decode base64 to Uint8List via `compute(base64Decode, base64Data)` (background isolate for large images). Upload to `bug-screenshots` bucket via `SupabaseConfig.client.storage.from('bug-screenshots').uploadBinary(path, bytes, fileOptions: FileOptions(contentType: 'image/png', upsert: true))`. Path: `'$reportId.png'`. Return `getPublicUrl(path)`. Wrap in try/catch — return null on failure (non-fatal).

`_createGitHubIssue(repo, title, body, token)` → Future<String> — POST to `https://api.github.com/repos/$repo/issues` with headers `Authorization: Bearer $token`, `Accept: application/vnd.github+json`, `X-GitHub-Api-Version: 2022-11-28`. Body: `{'title': title, 'body': body, 'labels': ['bug', 'bug-report']}`. Check 401 → throw `GitHubAuthException()`. Check != 201 → throw generic Exception. Return `json['html_url']` (NOT `json['url']` — that's the API URL, not the web URL).

Import `package:http/http.dart` as `http`, `package:crypto/crypto.dart`, `dart:convert`, `package:flutter/foundation.dart` (for compute).

- Method `Future<SyncResult> sync(String reportId, BugReportDetail detail)`:
  1. If `_isSyncing`, return `SyncError('Already syncing')`.
  2. Set `_isSyncing = true`, `_lastResult = null`, `notifyListeners()`.
  3. Call `_syncService.syncReport(reportId, detail)`.
  4. Set `_lastResult = result`.
  5. In `finally` block: `_isSyncing = false`, `notifyListeners()`.
  6. Return `result`.
